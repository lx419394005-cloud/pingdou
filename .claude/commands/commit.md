# /commit

将当前项目改动直接提交并推送到 GitHub，且提交信息必须是中文。

用法：
- `/commit`
- `/commit 修复画布平移与缩放交互`

执行规则：
1. 先检查工作区状态：`git status --short`。
2. 如果没有可提交改动，直接回复“当前没有可提交内容”，不要创建空提交。
3. 暂存所有改动：`git add .`。
4. 生成或使用中文提交信息：
   - 若用户在命令后提供了文本（`$ARGUMENTS`），直接作为提交标题使用。
   - 若未提供，基于本次改动生成一个简洁的中文标题（不超过 30 字）。
   - 禁止使用英文提交标题。
5. 提交：`git commit -m "<中文标题>"`。
6. 推送到主分支：`git push origin main`。
7. 最后回复：
   - 提交哈希（短哈希）
   - 提交标题
   - 推送结果

额外约束：
- 不要执行破坏性 git 操作（如 reset --hard、rebase）。
- 如推送失败，明确输出失败原因与下一步建议。
